Chakra Community Repository Tools (@ccr-tools)

The Chakra Community Repository tools are a collection of small apps and libraries to help users interact with the CCR. These tools include ccr, the CCR package installation script, ccr-tools, a small script for talking to CCR via CLI, meant mostly for package maintainers, python-ccr, a python library for interacting with the CCR, and aur2ccr, a program that automatically converts packages from AUR and the Arch repos to CCR source packages. You can install ccr and ccr-tools with pacman -S ccr, python-ccr with ccr -S python-ccr (NOTE: python-ccr will be added to the CCR when version 0.2 is complete. Until then, build it manually with the provided PKGBUILD.), and aur2ccr with ccr -S aur2ccr.

Example usage:

# Convert [package] from AUR or Arch repos to CCR package:
aur2ccr -m "My Name <email@example.com>" [package]
# Upload [package] to CCR:
ccr-tools --submit [package]/[package]/[package]-[pkgver]-[pkgrel].src.tar.gz
# Vote for [package] in CCR:
ccr-tools --vote [package]
# Install the package from CCR
ccr -S [package]



Searching:

# Search CCR for [package]:
ccr -Ss [package]
# Search AUR for [package]:
aur2ccr --search [package]
# Search for packages owned by [username] in CCR:
ccr-tools --maintainer [username]
# Searching with python-ccr:
>>> import ccr
>>> ccr.search('[package]')
>>> ccr.msearch('[username]')



More about the tools:


The ccr-tools repo includes the 'ccr' and 'ccr-tools' scripts, which are distrubuted in the Chakra repos as ccr. The first script, 'ccr', can be used to install user-created packages from the CCR. It also handles searching through CCR packages (ccr -Ss keyword) and updating installed CCR packages (ccr -Syu). It's companion script, 'ccr-tools' (formerly 'ccr-vote'), can be used to vote for packages in the CCR (ccr-tools --vote packagename), upload new packages to the CCR (ccr-tools --submit pkgname-pkgver-pkgrel.src.tar.gz), list CCR packages maintained by a given user (ccr-tools --maintainer username), and more. It is meant mainly for use by CCR package maintainers, as opposed to CCR users.

The 'ccr' script is a fork of 'packer', which was originally written by Matthew Bruenig. It is licensed under version 3 of the GPL. 'ccr-tools' was originally forked from 'aurvote', and is now maintained by Ram-Z and the other members of the ccr-tools team.


The python-ccr library allows easy access to a variety of functions for working with the Chakra Community Repository. Currently, it works with python2, but porting the lib to python3 is currently in progress. This library includes functions such as 'search', 'msearch', 'info', 'vote', 'flag', 'notify', and much more.

python-ccr is provided under version 2 of the GPL, or at your option, any later version. The python2 library was written and is currently maintained by george, Inkane, and the other members of the ccr-tools team. The python3 library is being written by stephenmac7 and the other members of the ccr-tools team.


aur2ccr is a small but flexible tool that will search AUR and the Arch repos for any package (aur2ccr packagename), download it if it exists, update the maintainer info in the PKGBUILD (when used with the -m/--maintainer option), and create and test a CCR-ready source archive. It also handles dependency checking, and will build a package for every dependency not in CCR or the Chakra repos. The script accepts several other options, so feel free to play around with it (or just read the aur2ccr(8) man page). If a package fails, try editing the PKGBUILD first, with the -e/--edit option.

To learn more about aur2ccr, see its forum thread. This script is available for download from the CCR, here. It is currently maintained by Inkane, george, and the other members of the ccr-tools team.




If you would like to file a bug report, request a feature, or contribute code, you can find each of these tools in their respective git repositories on GitHub, or their mirrors on Gitorious. The preferred method for reporting bugs or requesting features is to open an issue on GitHub (aur2ccr issues are here, python-ccr issues are here, and ccr-tools issues are here), but you can also ask for help on the Chakra forums.
